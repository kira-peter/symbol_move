# Project Context

## Purpose
符动世界 (SymbolMove) - 字符符号在动，创造世界。

这是一个用于创作有趣、酷炫功能的实验性项目。核心理念是用字符配合代码实现各种视觉效果和交互体验，包括但不限于：
- 字符动画和特效
- 终端艺术和可视化
- 创意编程实验
- 符号系统设计

## Tech Stack
- **Go** - 主要编程语言
- **终端/控制台** - 主要输出界面
- Git - 版本控制

## Project Conventions

### Code Style
- 遵循 Go 标准规范 (gofmt, golint)
- 命名约定：
  - 导出标识符使用 PascalCase
  - 未导出标识符使用 camelCase
  - 包名使用小写单词
- 注释可使用中文或英文，保持清晰
- 优先考虑代码可读性和趣味性

### Architecture Patterns
- **简单优先** - 避免过度设计
- **模块化设计** - 每个模块专注单一功能
- **可组合性** - 功能可以灵活组合使用
- **实验友好** - 易于尝试新想法和快速迭代

### Testing Strategy
- 使用 Go 标准测试框架
- 核心功能编写单元测试
- 视觉效果可通过示例程序验证
- 测试文件：`*_test.go`

### Git Workflow
- 主分支：main
- 提交信息清晰描述改动
- 小步提交，保持每次提交的原子性
- 功能开发遵循 OpenSpec 工作流

## Domain Context
**符动世界 (SymbolMove)** 的核心是"让字符动起来"：
- **字符** - 不仅是文本，更是构建视觉效果的基础元素
- **符号** - 包括 ASCII、Unicode、特殊字符等
- **动** - 时间维度上的变化、动画、交互
- **世界** - 用字符创造出丰富的视觉体验和交互环境

典型应用场景：
- 终端动画播放器
- ASCII/Unicode 艺术生成器
- 字符雨、矩阵效果等视觉特效
- 交互式终端游戏或可视化工具

## Important Constraints
- **实验性质** - 注重创意和趣味性，而非生产就绪
- **轻量级** - 尽量使用标准库，避免重依赖
- **终端友好** - 主要在命令行/终端环境运行
- **跨平台考虑** - Go 的跨平台特性，但终端特性可能有差异

## External Dependencies
- 主要依赖 Go 标准库
- 可能使用的第三方库：
  - 终端控制库 (如 termbox-go, tcell 等)
  - 颜色输出库 (如 fatih/color)
  - 根据需要按需引入，保持克制
