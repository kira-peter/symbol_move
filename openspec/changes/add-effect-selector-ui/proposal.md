# Change: 添加特效选择器主界面

## Why
为了提供更好的用户体验，需要一个统一的入口界面让用户浏览和选择不同的特效。当前每个特效都是独立的可执行文件，用户需要知道具体的命令才能运行，这不够直观和友好。

一个交互式的菜单界面可以：
- 让用户轻松发现所有可用的特效
- 提供统一、优雅的交互体验
- 方便未来添加更多特效
- 展示 SymbolMove 项目的完整能力

## What Changes
- 创建交互式特效选择器主界面
- 支持上下键浏览特效列表
- 高亮显示当前选中的特效
- 回车键确认并播放选中的特效
- ESC/q 键从特效返回主界面
- 提供特效描述信息预览
- 创建统一的主程序入口 `symbol-move`

## Impact
- 新增规格: `specs/effect-selector` - 特效选择器UI的规格
- 新增代码:
  - `pkg/ui/selector/` - 选择器组件实现
  - `pkg/effects/registry.go` - 特效注册表
  - `cmd/symbol-move/` - 新的主程序入口
- 修改代码:
  - `pkg/effects/matrix-rain/rain.go` - 重构为可被调用的形式
  - 添加特效生命周期管理（初始化、运行、清理）
- 用户体验: 从命令行参数驱动 → 交互式菜单驱动
